OOP object oriented programming

OOP is a design philosophy, everything in OOP is grouped as self sustainable objects

Where common properties and methods are grouped into a class. Then we can use the class (Blueprint) to create different objects or instances.

e.g. class could be vehicle , and subclasses could be Car, truck , bus. and objects or instances = bmw and mini
classes defines attributes (data) and behaviour (actions)
similar to CSS class property : value.
attributes can be called properties and behaviour called methods

OOP is where the code is split apart into self contained objects. almost like having smaller programs. each object represents a different part of the application. each object has its own logic and data.

OOP allow projects to be divided into object based program development.This is especially useful for large projects with many programmers can focus on individual objects

Objects can include new attributes and behaviours which allows it to be extensible

OOP allows objects to be used many times, hence reuseable.

Object Oriented Programming enables

  • Modularity ; self contained
  • Extensibility; can be modified
  • Reusability;can be used many times

Due to the above three factors, OOP has the follow five benefits

  1. easier to maintain code
  2. increased productivity
  3. faster to market
  4. lower costs
  5. more time to focus on quality of code

module is a python file which can contain many methods.we can import a module and use methods contained in the file.
